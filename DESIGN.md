# DESIGN.md

> UI/UX 원칙. 시각적 일관성과 인터랙션의 단일 진실 공급원.

## 디자인 철학

1. **모던하지만 정보 밀도 높음** — 빈 공간 낭비 없이, 깔끔하게
2. **키보드 1급 + 마우스 1급** — 둘 다 동등하게 효율적
3. **위험한 작업은 명시적, 안전한 작업은 무마찰** — 복사는 즉시, 영구 삭제는 단어 타이핑
4. **시스템 다크/라이트 모드 따르기** — 사용자 환경 존중
5. **TC 룩앤필 안 따라감** — 1990년대 디자인 탈출. 모던 데스크탑 앱처럼

### 영역별 역할

- **타이틀바**: 앱 이름, 글로벌 검색 (Ctrl+P), 윈도우 컨트롤
- **사이드바** (왼쪽): 로컬 / 호스트 목록 / 즐겨찾기. 토글 가능 (`Ctrl+B`)
- **메인 (듀얼 패널)**: 좌/우 패널. 활성 패널 강조
- **진행률 바**: 백그라운드 작업이 있을 때만 표시 (없으면 숨김)
- **상태바**: 선택 정보, 호스트 연결 상태

### 반응형

- 윈도우 폭 < 1100px: 사이드바 자동 숨김 (햄버거로 토글)
- 폭 < 800px: 한 패널만 표시 (좌/우 토글)
- 패널 폭 비율은 사용자가 드래그로 조정. 더블클릭하면 50/50 리셋

### 듀얼 패널 옵션

- **좌/우 (디폴트)**: 가로 분할
- **상/하**: `Ctrl+Shift+H` 토글
- **싱글**: 한 패널만 풀스크린 (`Ctrl+1`)

### 색상 쓰임

- **Accent (파랑)**: 활성 패널 보더, 선택, 포커스, 링크
- **Danger (빨강)**: 영구 삭제, 연결 실패, 에러
- **Muted (회색)**: 메타데이터 (사이즈, 시간), 비활성 텍스트
- **파일 타입 색상**: 매우 서브틀하게. 디렉토리만 살짝 강조, 나머지는 거의 default

**과한 색상 금지**: 한 화면에 강조색 4개 넘으면 디자인 실패.

## 컴포넌트 가이드

### 파일 리스트 (EntryList)

- **가상 스크롤** 필수 (`@tanstack/react-virtual`). 1만 항목도 즉각 응답
- 컬럼: 이름 / 크기 / 수정시간 / (옵션: 권한)
- 컬럼 헤더 클릭 = 정렬, 한 번 더 = 역순
- 디렉토리는 항상 위 (디폴트, 설정으로 변경 가능)
- 행 높이: 24px (조밀 모드) / 28px (보통) / 32px (편안)

### 행 (EntryRow)

```
┌──────────────────────────────────────────────────────┐
│ [□] [icon] name                  size      modified  │
└──────────────────────────────────────────────────────┘
  ↑    ↑    ↑                       ↑           ↑
 선택  타입  파일명                  크기         시간
```

상태별 표시:
- **호버**: bg-subtle
- **커서 (현재 포커스)**: 보더 좌측 2px accent
- **선택**: bg-active + 체크박스 채움
- **커서 + 선택**: 둘 다

### 아이콘

- **lucide-react** 사용 (이미 설치)
- 파일 타입별 아이콘:
  - 디렉토리: `Folder` / `FolderOpen`
  - 파일: `File` (확장자별 분기는 점진적으로 추가)
  - 심볼릭 링크: `Link`
  - 이미지/비디오/오디오: 해당 lucide 아이콘
- 아이콘 크기: 16px
- 색상: 디렉토리만 살짝 색상 (accent), 나머지는 muted

### 패널 헤더 (PathBar)

```
┌──────────────────────────────────────────────────────┐
│ [←] [→] [↑]  user@host  /  var  /  log  /  ▼  ⟳   │
└──────────────────────────────────────────────────────┘
  ↑           ↑                                ↑   ↑
 nav 버튼     호스트                          북마크 새로고침
```

- **breadcrumb 스타일** 경로. 각 세그먼트 클릭 = 그 디렉토리로 점프
- 호스트 표시 (로컬이면 "Local", SSH면 `user@host`)
- 직접 입력 모드 (Ctrl+L): breadcrumb → text input

### 사이드바 (왼쪽)

```
┌──────────────────┐
│  📁 Local         │  ← 로컬 (always)
├──────────────────┤
│  Hosts            │  ← 섹션 헤더
│  ● srv1.local     │  연결됨 (초록 점)
│  ○ srv2.local     │  연결 안 됨
│  ○ work-jump      │
├──────────────────┤
│  ⭐ Bookmarks      │
│  📌 ~/projects    │
│  📌 srv1:/var/log │
│  📌 ...           │
└──────────────────┘
```

- 호스트는 `~/.ssh/config` 자동 로드 + 사용자 추가 가능
- 더블클릭 = 활성 패널에 열기
- 우클릭 = 컨텍스트 메뉴 (다른 패널에 열기, 새 탭, 삭제)
- 드래그 = 즐겨찾기 재정렬

### 다이얼로그 패턴

#### 1. 즉시 액션 다이얼로그 (작은 모달)

이름 입력 등 짧은 인풋:

```
┌─ Rename ─────────────────────┐
│                              │
│   New name:                  │
│   ┌──────────────────────┐   │
│   │ document.txt        _│   │
│   └──────────────────────┘   │
│                              │
│              [Cancel] [OK]   │
└──────────────────────────────┘
```

- 입력 즉시 포커스
- Enter = 확인, Esc = 취소
- 폭 ~400px, 콘텐츠에 맞춰

#### 2. 확인 다이얼로그

복사/이동 충돌, 휴지통 삭제 등:

```
┌─ Delete to trash? ───────────────┐
│                                  │
│  Delete 3 items to trash:        │
│    • document.txt                │
│    • Photos/                     │
│    • backup.zip                  │
│                                  │
│  Total: 4.2 MB                   │
│                                  │
│       [Cancel]  [Delete]         │
└──────────────────────────────────┘
```

- "Delete" 버튼은 accent (파랑), 위험 아님 (휴지통이니까)

#### 3. 위험 확인 다이얼로그 (영구 삭제)

```
┌─ ⚠ Permanently delete? ───────────┐
│                                   │
│  This CANNOT be undone.           │
│                                   │
│  Permanently deleting:            │
│    • document.txt                 │
│    • Photos/  (1,234 files)       │
│                                   │
│  Type "delete" to confirm:        │
│  ┌─────────────────────────┐      │
│  │ _                       │      │
│  └─────────────────────────┘      │
│                                   │
│       [Cancel]  [Delete]          │
└───────────────────────────────────┘
```

- 빨간 보더
- "Delete" 버튼은 빨간색
- 단어 타이핑 강제 (실수로 Enter 못 누름)

#### 4. 진행률 다이얼로그

작은 작업은 진행률 바에 표시. 큰 작업 / 사용자 액션 필요한 경우 모달:

```
┌─ Copying files ──────────────────┐
│                                  │
│  foo.zip → /tmp/                 │
│  ████████████░░░░░░░  60%        │
│  142 MB / 235 MB · 8.2 MB/s      │
│  ETA: 12s                        │
│                                  │
│  [Background]    [Cancel]        │
└──────────────────────────────────┘
```

- "Background" = 닫고 진행률 바에서 계속 보기
- "Cancel" = 작업 취소

## 인터랙션 디테일

### 드래그 앤 드롭

- 패널 간 드래그 = 복사 (Ctrl 누르고 드래그 = 이동)
- 사이드바 즐겨찾기로 드래그 = 북마크 추가
- 외부에서 드래그 (브라우저 등) = 활성 패널에 업로드/저장
- 외부로 드래그 (Finder 등) = 외부에 다운로드

### 우클릭 메뉴

```
┌─────────────────────────┐
│ Open                    │
│ Open in other pane      │
│ ──────────────────────  │
│ Copy           Ctrl+C   │
│ Cut            Ctrl+X   │
│ Paste          Ctrl+V   │
│ ──────────────────────  │
│ Rename         F2       │
│ Delete         Del      │
│ ──────────────────────  │
│ Properties     Alt+Enter│
└─────────────────────────┘
```

### 진행 피드백

- **즉시**: 50ms 안에 끝나는 작업 (이름 변경 등) — 다이얼로그 없음, 결과만
- **짧은 작업**: 50ms ~ 2초 — 인디케이터 (커서 옆 스피너)
- **긴 작업**: 2초+ — 진행률 바에 자동 표시
- **매우 긴 작업**: 다이얼로그 모달, 백그라운드 가능

스피너: `⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏` (Tauri 윈도우는 트루컬러 OK니까 lucide의 `Loader2` 사용)

### 알림 (Toast)

- 우측 하단에 작은 토스트
- 자동 사라짐 (3초), 클릭하면 확장
- 에러는 사라지지 않음 (수동 닫기)

```
┌──────────────────────────┐
│ ✓ Copied 3 files         │
│   to /tmp/               │
└──────────────────────────┘
```

## 다크 모드 / 라이트 모드

- 시스템 설정 자동 감지 (`prefers-color-scheme`)
- 사용자 명시적 오버라이드 가능 (설정 또는 `Ctrl+Shift+T`)
- 모든 컴포넌트는 양쪽 모드에서 가독성 검증

## 절대 하지 말 것

- ❌ TC 룩앤필 모방 (보더 많고 색 많은 90년대 룩)
- ❌ 정보 빽빽하게 쌓기 (적절한 패딩 필수)
- ❌ 한 화면에 강조색 4개 이상
- ❌ 애니메이션 남발 (200ms 넘으면 짜증남)
- ❌ 토스트 알림 남발 (성공 알림은 큰 작업만)
- ❌ 키보드 단축키 메모리에 의존 (커맨드 팔레트로 모든 명령 접근)
- ❌ 다크 모드 무시
- ❌ 크기 작은 클릭 타겟 (최소 24px)
- ❌ 파괴적 작업이 실수로 일어나기 쉬운 UI (영구 삭제는 단어 타이핑)
- ❌ 진행률 표시 없는 장기 작업
- ❌ 같은 호스트 SFTP 복사가 본인 PC를 거쳐가는 폴백 (이게 핵심 가치)

| 윈도우 컨트롤 | 우상단 (— □ ✕) | 좌상단 (✕ — +) | DE 따라 |
| 메뉴바 | 앱 내부 | 시스템 메뉴바도 | 앱 내부 |
| 폰트 | Segoe UI | SF Pro | Inter / system |
| 단축키 | Ctrl | Cmd | Ctrl |
| 휴지통 표현 | "Recycle Bin" | "Trash" | "Trash" |
| 경로 표시 | `C:\Users\name` | `/Users/name` (`~`) | `/home/name` (`~`) |
| 드라이브 | `C:`, `D:` 사이드바 | `/Volumes/*` | `/mnt/*` |

Tauri 자체가 OS별 윈도우 컨트롤은 알아서 처리. 우리는 콘텐츠만.
